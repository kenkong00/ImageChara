# 角色数据编辑与注入功能计划

## 需求概述
当前项目可以解析 PNG 图片中的角色数据（`chara` tEXt chunk，base64编码的JSON），但只能**只读查看**。需要新增：
1. **编辑功能**：在角色 Tab 中可编辑角色数据（包括无角色数据的图片也可新建）
2. **保存功能**：将编辑后的角色数据注入回 PNG 图片并下载

## 现有架构分析

### 角色数据存储方式（PNG）
- 存储位置：PNG 的 `tEXt` chunk，keyword 为 `chara`
- 编码格式：`Base64(JSON)`
- 读取逻辑：[metadataParser.js L442-L492](web/src/utils/metadataParser.js#L442-L492) — `extractCharacterFromPNG()`

### 角色数据字段结构（从模板推断）
```json
{
  "name": "角色名称",
  "description": "角色描述",
  // ... 其他自定义属性字段
  "data": { ... }  // 扩展数据
}
```

### 当前角色 Tab 展示
- [App.vue L100-L145](web/src/App.vue#L100-L145)：展示 name、description、attributes、data、raw JSON
- 无角色数据时显示占位提示

## 实施步骤

### 步骤 1：创建角色数据写入工具函数
**文件**: `web/src/utils/metadataWriter.js`（新文件）

实现 `injectCharacterToPNG(uint8Array, charaData)` 函数：
- 输入：原始图片的 `Uint8Array` + 角色 JSON 对象
- 处理逻辑：
  1. 遍历 PNG chunks，查找已有的 `chara` tEXt chunk
  2. 如果找到 → 替换其值（保持原 chunk 长度或重新计算）
  3. 如果没找到 → 在 IEND 前插入新的 tEXt chunk
  4. 将 charaData JSON → UTF-8 → Base64 编码为 chunk value
- 输出：修改后的 `Uint8Array`
- 同时导出 `jsonToBase64()` 工具函数（与 parser 中的 `base64ToJson()` 对称）

### 步骤 4：保存/下载逻辑
**文件**: `web/src/App.vue` 脚本部分

`saveCharaToImage()` 流程：
1. 从 `currentImage`（base64 DataURL）还原为 ArrayBuffer
2. 调用 `injectCharacterToPNG(arrayBuffer, editCharaData.value)` 得到新 Uint8Array
3. 创建 `new Blob([newUint8Array], { type: 'image/png' })`
4. 用 `URL.createObjectURL()` + `<a>` 标签触发下载，文件名格式：`原文件名_chara.png`
5. 设置 `saveStatus = 'success'`，2 秒后复位
6. 自动退出编辑模式，重新解析更新后的元数据（如果用户选择了新文件）

## 技术要点

### PNG 二进制操作注意事项
- PNG chunk 结构：`[4字节长度][4字节类型][数据][4字节CRC]`
- tEXt chunk 数据格式：`keyword\0value`（keyword 必须为 ASCII）
- 修改后需重新计算 CRC32（需实现 CRC32 函数）
- 在 IEND chunk 之前插入新 chunk 是最安全的方案

### Base64 编码（写入方向）
```js
// JSON → UTF-8 bytes → Base64 string（与 base64ToJson 对称的逆操作）
function jsonToBase64(obj) {
  const jsonStr = JSON.stringify(obj)
  const encoder = new TextEncoder()
  const bytes = encoder.encode(jsonStr)
  return btoa(String.fromCharCode(...bytes))
}
```

### 不使用 Canvas 重编码的原因
Canvas 重编码会改变图片压缩率和像素数据（尤其有 alpha 通道时），直接操作 PNG 二进制可以**无损保留原图质量**。
